Intersection Detection Acceleration Structure

Posted on 2020-09-18 Edited on 2020-10-03

Introduction

Ray tracing is the main implementation method of offline rendering. It generates a real picture by simulating the path of light propagation. However, Ray tracing is very inefficient due to we need to cast thousands of rays, and test their intersections with scene objects. To reduce the rendering time, we need to find some superior spatial acceleration structures. This project will explore some of the popular acceleration data structures, and implement a ray tracing scene for performance comparison. In addition, this project will also compare their applicable scenarios, advantages, disadvantages, and possible optimization solutions.

Background Information

The naive approach to this problem would be for each ray, do a collision test with every object. This would result in $O(n)$ time for each ray. When a lot of rays are emitted, the rendering efficiency will drop sharply.
There are many ways to reduce this complexity, but this project will focus on the method of computation geometry. An efficient ray tracing implementation needs to rely on an acceleration structure to take advantage of the locality of objects. The main principle behind these structures is to safely reject or ignore whole groups of primitives, which cannot be intersected by a given ray.

Generally, there are two types of acceleration structure: grid-based and tree-based.

  • Grid-Based structure refers to dividing the space into grids of fixed size, and then do intersection detections based on which grid the objects and rays are located.
  • Tree-Based structure instead utilize the hierarchical structure of the objects, determine which object to collide by subdividing from outside to inside, from top to bottom. Among the most representative data structures are KD trees, bounding volume hierarchies and octrees.

This project will briefly introduce the grid-based method, and the main focus will be on BVH and KD trees. They are currently the best acceleration structure in collision detection.

Input and Output

I will prepare several different 3D ray tracing scenes, and test their respective rendering time. When performing ray tracing, I intended to use a recursive ray tracing method. When the ray hits the surface of an object, it will reflect according to the properties of the surface recursively, but I limit the maximum number of times the ray bounces back and forth to 5. I did this on purpose to allow light to hit the object from various angles, not just from the perspective of the viewport.
